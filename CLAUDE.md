## Commands

```bash
# Install dependencies (run from root)
pnpm install

# Run web frontend in browser (hot reload)
pnpm --filter @auto-shutdown/web dev

# Run desktop app in Wails dev mode (starts Go + frontend watcher)
cd apps/desktop && wails dev
# or from root:
pnpm wails

# Build desktop executable вҶ’ apps/desktop/build/bin/auto-shutdown.exe
cd apps/desktop && wails build
# or from root:
pnpm wails-build

# Lint frontend
pnpm --filter @auto-shutdown/web lint

# Run all dev/build/lint via Turborepo
pnpm dev
pnpm build
pnpm lint
```

## Architecture

This is a **pnpm + Turborepo monorepo** for a Windows desktop app that auto-shuts down when network/disk activity drops below configurable thresholds.

### Workspaces

| Path | Package | Role |
|------|---------|------|
| `apps/web` | `@auto-shutdown/web` | React UI (also serves as Wails frontend) |
| `apps/desktop` | `@auto-shutdown/desktop` | Go + Wails v2 desktop host |
| `packages/config` | `@auto-shutdown/config` | Shared TS/ESLint/Tailwind config |

### Frontend (`apps/web`)

- **React 19** + **Vite 8** + **TailwindCSS v4** + **TanStack Router v1**
- Uses `createMemoryHistory` (not browser/hash history) вҖ” required because Wails embeds the app in a WebView2 window, not a real browser.
- Routes are file-based under `src/routes/`; `routeTree.gen.ts` is auto-generated by the TanStack Router Vite plugin вҖ” do not edit it manually.
- Global state lives in **Zustand** stores under `src/store/`. The main store is `monitor.ts`, which mirrors the Go engine's state machine.
- UI components follow shadcn conventions in `src/components/ui/`.

### Desktop (`apps/desktop`)

- **Wails v2** bridges Go to the frontend. All public methods on `App` (in `app.go`) are automatically bound and callable from the frontend as `window.go.main.App.*`.
- The Wails config (`wails.json`) points `frontend:dev:serverUrl` at `http://localhost:5173`, so `wails dev` proxies to the Vite dev server.
- For production builds, `wails build` runs `pnpm --filter @auto-shutdown/web build` first, then embeds `frontend/dist` via Go's `//go:embed` directive in `main.go`.

### Go Backend (`internal/`)

Four packages implement the core logic вҖ” none are wired into `app.go` yet (marked TODO):

- **`internal/monitor`** вҖ” samples network + disk I/O using `gopsutil`. Returns `Metrics{NetworkKbps, DiskMBps}` as delta from previous sample.
- **`internal/engine`** вҖ” pure state machine with no I/O. States: `ACTIVE вҶ’ IDLE_CANDIDATE вҶ’ CONFIRMED_IDLE вҶ’ COUNTDOWN вҶ’ SHUTDOWN`. Call `Tick(networkKbps, diskMBps)` each sample interval; call `Cancel()` to abort a countdown; call `ConfirmShutdown()` to trigger terminal state.
- **`internal/executor`** вҖ” executes the OS power action (`shutdown`, `sleep`, `hibernate`) via Windows shell commands.
- **`internal/config`** вҖ” loads/saves `Settings` as JSON to `%APPDATA%\auto-shutdown\config.json`; returns `DefaultSettings()` if the file doesn't exist.

### Prerequisites

- Node.js вүҘ 18, pnpm вүҘ 8
- Go вүҘ 1.21
- Wails v2: `go install github.com/wailsapp/wails/v2/cmd/wails@latest`
- WebView2 Runtime (Windows вҖ” usually pre-installed)
